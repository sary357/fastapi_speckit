# Data Model: Like/Dislike 與留言收集 API

**Feature**: [spec.md](./spec.md) | **Plan**: [plan.md](./plan.md)

## 實體總覽

本功能包含兩個彼此獨立、互不關聯的資料實體，對應規格 Key Entities 段落。

### 1. ReactionRecord（反應紀錄）

代表一次使用者送出的「讚」或「不讚」事件。

| 欄位 | 型別 | 說明 | 驗證規則 |
|---|---|---|---|
| `id` | UUID / 遞增整數（實作時擇一） | 主鍵，唯一識別一筆反應紀錄 | 系統自動產生 |
| `reaction_type` | Enum（`like` \| `dislike`） | 反應類型 | 必填；僅允許 `like` 或 `dislike` 兩種值（FR-001, FR-002） |
| `received_at` | `timestamptz`（UTC） | 系統收到請求的時間 | 必填；由伺服器於收到請求當下以 UTC 賦值（FR-004） |
| `source_ip` | 字串（支援 IPv4／IPv6） | 送出請求的來源 IP | 必填；由伺服器於收到請求當下擷取（FR-004） |

**狀態轉換**：無。每筆紀錄建立後即為不可變的歷史事件，本版本不提供更新或刪除操作
（對齊規格 Assumptions：不提供編輯或刪除既有紀錄的機制）。

### 2. CommentRecord（留言紀錄）

代表一次使用者送出的留言事件。

| 欄位 | 型別 | 說明 | 驗證規則 |
|---|---|---|---|
| `id` | UUID / 遞增整數（實作時擇一） | 主鍵，唯一識別一筆留言紀錄 | 系統自動產生 |
| `content` | 文字（TEXT） | 留言內容 | 必填；去除前後空白後長度需 ≥ 1（不得為空白或缺漏，FR-008） |
| `received_at` | `timestamptz`（UTC） | 系統收到請求的時間 | 必填；由伺服器於收到請求當下以 UTC 賦值（FR-007） |
| `source_ip` | 字串（支援 IPv4／IPv6） | 送出請求的來源 IP | 必填；由伺服器於收到請求當下擷取（FR-007） |

**狀態轉換**：無。每筆紀錄建立後即為不可變的歷史事件，本版本不提供更新或刪除操作。

## 關聯關係

`ReactionRecord` 與 `CommentRecord` 彼此獨立，不與任何特定內容項目（如文章、貼文）或
使用者帳號建立關聯（對齊規格 Assumptions）。兩個實體之間也沒有直接關聯。

## 資料保留與刪除

依規格 Assumptions，資料將長期保存，本版本不包含自動到期、封存或清除機制；因此本資料
模型不包含到期時間（TTL）或軟刪除（soft-delete）欄位。

## 個資考量

`source_ip` 欄位可能構成個人資料（PII）。依憲章 VIII（安全標準）與規格 Assumptions，
本版本僅記錄不做進一步分析或第三方分享；正式合規（如 GDPR/CCPA 的保留期限、當事人請求
刪除等）留待未來版本依實際法遵需求評估與實作，本次不在規格與計畫範圍內。
